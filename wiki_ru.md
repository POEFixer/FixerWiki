# POEFixer Wiki

Добро пожаловать в документацию проекта POEFixer. Ниже приведено описание функционала программы, разбитое по вкладкам интерфейса.

---

## 1. Login (Браузер Торговли)

Вкладка **Login** — это отправная точка для работы с торговлей. Она интегрирует полноценный веб-браузер прямо в приложение, позволяя взаимодействовать с официальным сайтом Path of Exile Trade, не переключаясь между окнами.

### Основные возможности

#### Встроенный веб-браузер

Браузер работает на базе **WebView2**, обеспечивая опыт, идентичный использованию Chrome или Edge.

- **Управление сессией:** Приложение автоматически перехватывает ваши cookies сессии. Это необходимо для того, чтобы вкладка **Trade** могла подключаться к официальному WebSocket API и получать обновления в реальном времени.
- **Поддержка режимов игры:** Автоматически определяет и поддерживает торговые сайты как для Path of Exile 1, так и для Path of Exile 2 в зависимости от ваших настроек.
- **Навигация:** Включает адресную строку для ручного ввода URL и кнопку **Home** для быстрого возврата на главную страницу поиска.

#### Добавление торговых фильтров

После настройки поиска на официальном сайте (например, "Headhunter" с определенными характеристиками), вы можете добавить его в систему мониторинга приложения.

1. **Настройте поиск:** Используйте браузер для выбора фильтров на сайте.
2. **Нажмите "Add":** Нажмите большую кнопку **Add** в верхней части вкладки.
3. **Проверка:** Инструмент проверит, является ли URL допустимым и соответствует ли он текущему режиму игры (PoE 1 или PoE 2).
4. **Подтверждение:** Появится диалоговое окно, позволяющее задать имя для поиска перед тем, как он начнет отслеживаться во вкладке **Trade**.

> **Примечание:** Если вы видите ошибку "WebView2 Missing", приложение предоставит ссылку для загрузки необходимой среды выполнения от Microsoft.

## 2. Trade (Торговля)

Вкладка **Trade** — это центр управления мониторингом рынка в реальном времени. Она использует официальный WebSocket API Path of Exile для получения мгновенных уведомлений о новых лотах, соответствующих вашим критериям поиска.

### Обзор Интерфейса

В основном окне отображается список всех ваших активных подключений (поисковых запросов).

*(Screenshot: Trade Tab Main View)*

#### Столбцы Списка Подключений

1.  **Status (Статус):**
    -   **Зеленый:** Подключено, ожидание новых предметов.
    -   **Оранжевый:** Подключение...
    -   **Красный:** Отключено.
    -   **Rate Limit:** Если API ограничило частоту запросов, здесь появится таймер обратного отсчета до возможности повторного подключения.
2.  **Name (Имя):** Название вашего поиска (например, "Headhunter").
    -   **Клик:** Открывает URL поиска в браузере по умолчанию.
    -   **Наведение:** Показывает полный URL во всплывающей подсказке.
3.  **Statistics (Статистика):**
    -   **Иконка Рукопожатия:** Количество найденных предметов за текущую сессию.
    -   **Галочка (Check):** Количество успешно отправленных сообщений о покупке.
    -   **Крестик (X):** Количество неудачных попыток.
4.  **Actions (Действия):**
    -   **Кнопка Play/Stop:** Вручную запустить или остановить живой поиск для этого конкретного подключения.
    -   **Shopping Cart (Корзина):** Открывает меню **Manual Buy** для массовой скупки предметов из этого поиска (В разработке).
    -   **Pen (Редактировать):** Изменить имя или URL подключения.
    -   **Gear (Настройки):** Открывает подробное окно **Настроек и Логов** для этого подключения.
    -   **Trash (Удалить):** Навсегда удалить это подключение.

### Глобальное Управление

В нижней части вкладки находятся глобальные кнопки управления:

-   **Connect All:** Последовательно подключается ко всем активным поискам. Добавляет небольшую задержку между подключениями, чтобы избежать блокировки API (Rate Limit).
-   **Disconnect All:** Немедленно останавливает все активные поиски.
-   **Add Link:** Позволяет вручную добавить URL торгового поиска, если вы не хотите использовать браузер во вкладке **Login**.

### Настройки Подключения и Логи

Нажатие на иконку **Шестеренки** открывает подробное модальное окно.

*(Screenshot: Connection Settings Window)*

#### 1. Общие Настройки (General Settings)

-   **Auto Reconnect:** Если включено, инструмент будет автоматически пытаться переподключиться при разрыве соединения.
-   **Enable Notification:** Показывает всплывающее уведомление поверх игры, когда найден новый предмет.
-   **Multi-Item Whisper Delay:** Добавляет настраиваемую задержку (в миллисекундах) между отправкой сообщений. Это критически важно для популярных поисков, чтобы избежать блокировки за спам.

#### 2. Фильтры Валют (Currency Filters)

Фильтрация входящих уведомлений по цене лота. Позволяет игнорировать слишком дешевые или дорогие лоты, не меняя фильтры на сайте.

-   **Поддержка PoE 1 / PoE 2:** Список валют автоматически адаптируется под выбранный режим игры.
-   **Min/Max:** Установите минимальное и максимальное количество для конкретных валют (например, уведомлять только о предметах ценой от 100 до 150 Chaos Orbs).

#### 3. Логи Торговли (Trade Logs)

В правой части окна настроек отображается история предметов, найденных именно этим поиском.

*(Screenshot: Trade Logs View)*

-   **Карточка Предмета:** Отображает иконку, название и цену лота.
-   **Информация о Продавце:** Имя игрока и время, когда предмет был выставлен.
-   **Моды:** Список свойств (аффиксов) предмета.
-   **Индикаторы Статуса:**
    -   **Travel:** Успешный переход в убежище продавца.
    -   **Buy:** Успешное копирование/отправка сообщения о покупке.

## 3. Settings (Настройки)

Вкладка **Settings** — это центр управления глобальным поведением приложения, интеграцией с игрой и настройками пользовательского интерфейса.

### Game Selection (Выбор игры)

- **Path of Exile 2 / Path of Exile 1:** Выберите режим игры, в который вы играете. Это изменяет URL-адреса для поиска торговли, логику обнаружения окна и совместимость сканеров.
- **League (Лига):** Выберите текущую лигу (например, Standard, Hardcore или временные лиги) из выпадающего списка, чтобы поиск торговли осуществлялся в правильной экономике.

### Scanner Type (Тип сканера)

Выберите способ, которым приложение считывает данные из игрового клиента.

- **WGC (Windows Graphics Capture):**
  -   **Описание:** Визуально захватывает окно игры, даже если оно находится в фоне. Рекомендуется для PoE 1 или если Memory Scanner недоступен.
  -   **Corner Density:** Настройка порога анализа кадров (расширенная настройка).
  -   **Use Full Window (Использовать всё окно):** Сканирует всё окно игры. Если отключено, можно вручную задать конкретную область экрана для сканирования (полезно для ультрашироких мониторов).
  -   **Select Area (Выбрать область):** Интерактивный инструмент для выделения области сканирования на экране.
- **Memory Scanner (Только для PoE 2):**
  -   **Описание:** Напрямую считывает память игры для идентификации предметов и сущностей. Обычно быстрее и точнее для PoE 2.

### Click Settings (Настройки кликов)

Настройка взаимодействия приложения с мышью, в основном для автоматической торговли или управления инвентарем.

- **Enable Auto Click:** Главный переключатель, разрешающий приложению выполнять клики мышью.
- **Click Count:** Количество кликов за одно действие (если в инвентаре нету валюты предмет может не выкупиться с первого раза).
- **Click Interval:** Задержка (в миллисекундах) между кликами.
- **Scan Timeout:** Максимальное время (в секундах), которое сканер будет пытаться найти предмет.
- **Hold Ctrl:** Автоматически удерживает клавишу `Ctrl` во время кликов (полезно для быстрого перемещения предметов).
- **Return Cursor:** Возвращает курсор мыши в исходное положение после выполнения действия.
- **Block Input:** Временно блокирует ввод мыши пользователя во время выполнения автоматического действия для предотвращения помех.
- **Enable Rate Limit:** Ограничивает частоту действий для предотвращения блокировок со стороны сервера.
- **Fast Trade:** Оптимизирует настройки для быстрой торговли.
- **Fast Mouse Click:** Уменьшает задержку между нажатием и отпусканием кнопки мыши.

### Advanced (Дополнительно)

- **Return to Hideout:** Автоматически пишет `/hideout` в чат после определенных действий (если настроено).
- **Streamer Mode (Режим стримера):** Скрывает оверлей и окна приложения от программ для стриминга (например, OBS), чтобы зрители видели только игру.

### Desktop Notifications (Уведомления на рабочем столе)

Настройка всплывающих уведомлений, которые появляются при нахождении предмета в поиске.

- **Enable Desktop Notifications:** Включить/выключить системные уведомления.
- **Position:** Выбор места появления уведомлений на экране (Сверху-справа, Сверху-слева и т.д.).
- **Display Time:** Время отображения уведомления (в секундах).
- **Enable Sound:** Проигрывать звук при появлении уведомления.
- **Display Options (Опции отображения):**
  -   **Show Icon:** Показывать иконку предмета.
  -   **Show Price:** Показывать цену.
  -   **Show Seller:** Показывать имя продавца.
  -   **Show Mods:** Показывать список модов предмета.
- **Test Notification:** Отправить тестовое уведомление для проверки настроек.

### Тема и Оформление

- **Тема:** Выбор из 4 визуальных тем для всего приложения:
  - **Dark** — Тёмная тема по умолчанию с мягким контрастом.
  - **Light** — Светлая тема для хорошо освещённых помещений.
  - **Blue** — Тёмная тема с синими акцентами.
  - **Midnight** — Глубокая тёмная тема с минимальной яркостью.
- **Масштаб шрифта:** Настройка глобального размера текста (0.8x — 1.5x) для комфортной читаемости на экранах с высоким разрешением.

### Language & Interface (Язык и Интерфейс)

- **Language:** Переключение языка интерфейса приложения (Английский, Русский, Корейский и др.). Поддерживается 11 языков.
- Все строки, связанные с темами, локализованы на всех поддерживаемых языках.

## 4. AutoPot (Автопоты) [PoE 2]

Вкладка **AutoPot** предоставляет функционал автоматического использования флаконов и вспомогательные утилиты специально для Path of Exile 2. Она включает в себя комплексную систему управления здоровьем (HP), энергетическим щитом (ES) и маной (MP) персонажа.

### Основные возможности

#### 1. Статус Устройства Ввода и Драйвера

Система использует виртуальный контроллер для безопасной эмуляции нажатий.

- **ViGEmBus Driver:** Проверяет наличие установленного драйвера ViGEmBus и предоставляет ссылку на скачивание, если он отсутствует.
- **Обнаружение Контроллера:** Автоматически определяет подключенные физические геймпады и активные виртуальные контроллеры.
- **Предупреждения:** Уведомляет, если назначены кнопки геймпада, но сам контроллер не обнаружен.

#### 2. Оверлей Жизненных Показателей (Vitals Overlay)

Настраиваемый визуальный HUD, отображающий текущие HP, ES и MP поверх игры.

- **Режимы Отображения:** 7 различных стилей на выбор:
  - *Horizontal Bars* (Классические горизонтальные полосы)
  - *Vertical Bars* (Компактные вертикальные полосы)
  - *Percent Text* (Простое текстовое отображение процентов)
  - *Circles* (Круговые индикаторы)
  - *Compact Blocks* (Минималистичные блоки)
  - *Minimal Dots* (Ненавязчивые точки)
  - *Glass Panel* (Стеклянная панель с прозрачным фоном)
- **Кастомизация:**
  - **Масштаб:** Регулировка размера оверлея (от 0.5x до 3.0x).
  - **Видимость:** Возможность отдельно включить/выключить отображение HP, ES и MP.
  - **Числовые Значения:** Переключение между процентами и абсолютными значениями.
  - **Перемещение:** Когда меню открыто, появляется подсказка "Drag Hint", позволяющая перетащить оверлей в любое удобное место экрана.

#### 3. Режимы Автоматизации

Выберите логику, подходящую под билд вашего персонажа:

- **Life:** Мониторинг только здоровья (HP).
- **Energy Shield (ES):** Мониторинг только энергощита.
- **Life + ES:** Одновременный мониторинг HP и ES.
- **Mind Over Matter (MOM):** Мониторинг маны (MP), часто используемой для поглощения урона.
- **MOM + Life:** Мониторинг MP и HP.
- **MOM + ES:** Мониторинг MP и ES.
- **All:** Мониторинг HP, ES и MP одновременно.

#### 4. Настройка Порогов (Thresholds)

Точный контроль условий использования флаконов.

- **Слайдеры:** Установка процентного порога (1-100%) для HP, ES и MP. Если значение падает ниже этого уровня, срабатывает соответствующая клавиша.
- **Умное отображение:** Показываются только те слайдеры, которые актуальны для выбранного **Режима**, чтобы не загромождать интерфейс.

#### 5. Авто-Дисконнект (Auto Disconnect / Chicken)

Функция безопасности для автоматического выхода из игры (логаута) при критическом снижении показателей.

- **Раздельные Триггеры:** Независимые переключатели и пороги срабатывания для HP, ES и MP.
- **Визуализация:** Пороги отображаются на Оверлее Жизненных Показателей в виде пунктирных линий или меток, показывая, когда именно произойдет выход.

#### 6. Привязка Клавиш и Задержки

Настройка клавиш для каждого действия.

- **Key 1:** Основные клавиши для HP или ES (в зависимости от режима).
- **Key 2:** Основные клавиши для MP или вторичного флакона.
- **Key 3:** Выделенная клавиша для ES (доступна только в режиме "All").
- **Типы Ввода:** Поддержка как клавиш клавиатуры, так и кнопок геймпада.
- **Задержки:** Установка кулдауна (в миллисекундах) для каждой клавиши, чтобы предотвратить слишком частое использование (спам).

## 5. Radar (Радар) [PoE 2]

Продвинутая система визуальной помощи и наложения карты, разработанная специально для Path of Exile 2. Она помогает в навигации, отслеживании сущностей и фильтрации.

### Ключевые особенности

#### 1. Общие настройки (General Settings)

Настройка основного поведения радара.

- **Draw Walkable Map (Отображать проходимую карту):** Включает визуализацию навигационной сетки игры (проходимой местности). Полезно для видимости доступных путей в темных областях.
  - **Color (Цвет):** Настройка цвета наложения карты.
  - **Border Thickness (Толщина границ):** Настройка толщины контуров границ карты.
- **Map Fix:** Включает режим коррекции для устранения проблем с выравниванием карты.
- **Логика видимости:**
  - **Hide in Town/Hideout:** Автоматически отключает радар в безопасных зонах для уменьшения визуального шума.
  - **Hide in Background:** Скрывает оверлей, когда окно игры не в фокусе (alt-tab).
  - **Hide on Pause:** Скрывает оверлей, когда игра на паузе.
  - **Hide Outside Network Bubble:** Скрывает сущности, находящиеся за пределами радиуса обновления сервера, чтобы предотвратить появление "призраков" (устаревших данных).
- **Show Player Names:** Включает отображение имен над игроками.

*(Screenshot: Radar General Settings)*

#### 2. Иконки (Icons)

Настройка визуального представления сущностей на радаре.

- **Группы:** Сущности разделены на группы (Base, League, Breach, Delirium, Expedition, etc.).
- **Кастомизация:** Для каждой группы можно назначить определенные иконки и настроить их размер. Это позволяет с первого взгляда отличать обычного монстра, босса или специфическую механику лиги.

*(Screenshot: Radar Icon Settings)*

#### 3. Поиск пути (Path Finder)

Мощный инструмент навигации, который рисует линии/пути к определенным целям.

- **Enable Path Finder:** Включает систему отрисовки путей.
- **Multicolor:** Рандомизирует цвета путей для лучшего различения нескольких целей.
- **Important POI:** Включает отображение "Точек интереса" (например, цели квестов или вейпоинты) с опциональным фоном текста.
- **Управление целями:**
  - **Add Target:** Ручное добавление цели по имени.
  - **Add from Map:** Вход в интерактивный режим для клика по объекту на радаре, чтобы добавить его в качестве цели.
  - **Add to Global/Area:** Выбор того, должна ли цель отслеживаться во всех зонах (Global) или только в текущей (Area).
- **Ignored Areas:** Вы можете отключить Path Finder в определенных зонах, чтобы избежать перегруженности.

*(Screenshot: Path Finder Tab)*

#### 4. Фильтры (Filters)

Тонкая настройка того, какие сущности отображаются на радаре и как они выглядят.

- **Add Nearby Entities:** Выпадающий список, который сканирует текущую память на наличие допустимых сущностей рядом с игроком (Монстры, Сундуки, NPC и т.д.). Выберите сущность и нажмите "Add", чтобы мгновенно создать фильтр для нее.
- **Manual Input:** Для продвинутых пользователей, позволяет добавлять фильтры, вручную вводя путь метаданных.
- **Режим редактирования:** Нажмите кнопку "E" рядом с любым фильтром, чтобы открыть подробный редактор:
  - **Name:** Задать пользовательское отображаемое имя.
  - **Color:** Выбрать пользовательский цвет текста.
  - **Icon:** Включить иконку, выбрать конкретное изображение и настроить размер.

  - **Ignore:** Полностью скрыть эту сущность с радара.

## 6. Smoother (Сглаживание)

Вкладка **Smoother** — это комплексный инструмент для оптимизации производительности игры и настройки визуальных элементов. Он работает путем модификации игровых файлов (например, `.ggpk`) для отключения или изменения конкретных графических эффектов.

### Основные возможности

#### Выбор игры и настройка

- **Версия игры:** Поддержка как Path of Exile 1, так и Path of Exile 2.
- **Путь к файлам:** Необходимо указать путь к файлу `Content.ggpk` (или папке `Bundles2` для PoE 2). Встроенный браузер файлов и система валидации помогут выбрать верный путь.
- **Система резервного копирования:**
  - **Создание бэкапа:** Автоматически создает резервные копии оригинальных файлов перед применением любых патчей.
  - **Restore All:** Кнопка для полного восстановления игры в исходное состояние (отмена всех изменений).
  - **Clear Backup:** Удаляет сохраненные бэкапы и хеши (полезно, если вы хотите начать с чистого листа).
  - **Обновления игры:** Программа автоматически определяет, была ли игра обновлена, и предлагает создать новый бэкап, чтобы избежать повреждения файлов.

#### Патчи производительности

Включайте отдельные патчи для отключения конкретных графических эффектов. Это может значительно повысить FPS на слабых системах.

- **Fog / Atlas Fog:** Удаляет туман с карт и Атласа.
- **Shadows & Lights:** Отключает динамические тени и расчеты освещения.
- **Particles:** Удаляет частицы окружения (дождь, пыль) и общие эффекты частиц.
- **Corpses:** Предотвращает отрисовку трупов (только PoE 1).
- **Delirium:** Удаляет визуальный туман и оверлей лиги Делириум.
- **EpkRemoval:** Удаляет пакеты эффектов для более чистого изображения.
- **Camera:** Позволяет отдалить камеру дальше стандартных лимитов (настраивается ползунком).

*(Screenshot: Список доступных патчей)*

#### Инструменты продвинутой настройки

##### 1. Color Mods (Цветовые модификации)

Настройка цветов текста для модификаторов предметов и характеристик.

- **Поиск и добавление:** Ищите конкретные статы (например, "Physical Damage") и назначайте им свой цвет.
- **Пресеты:** Включает готовые пресеты для опасных модов (Reflect, No Regen) или ценного дропа.
- **Экспорт/Импорт:** Делитесь своими конфигами с другими через JSON файлы.
- **Редактирование описаний:** Вы можете переписать текст описания модов для большей ясности.

*(Screenshot: Окно настройки Color Mods)*

##### 2. Spell Effects (Эффекты умений)

Мощный редактор на основе древовидной структуры для отключения эффектов конкретных умений.

- **Древовидный просмотр:** Навигация по всем эффектам игры, организованным по категориям.
- **Выборочное отключение:** Отключайте эффекты для конкретных скиллов (например, "Discharge", "Tornado Shot"), чтобы уменьшить визуальный шум, не удаляя всё подряд.
- **Поиск:** Быстрый поиск эффектов по названию.

*(Screenshot: Редактор дерева Spell Effects)*

##### 3. Black Screen (Режим черного экрана)

Режим экстремальной оптимизации, который удаляет текстуры и шейдеры, делая игровой мир почти черным, но оставляя видимыми важные элементы геймплея.

- **Расширения файлов:** Выбор типов файлов для игнорирования (например, текстуры `.dds`, звуки `.wav`).
- **Блоки анимации:** Отключение категорий анимации, таких как ParticleEffects, Lights или WindEvents.
- **Настройки SkinMesh:**
  - **Список исключений:** Выберите конкретные 3D модели (metadata), которые должны остаться видимыми даже в режиме Black Screen. Это критически важно для того, чтобы видеть своего персонажа, боссов или важные механики.
  - **Сканирование:** Инструмент сканирует файлы игры, чтобы составить полный список доступных моделей для выбора.

## 7. AutoCraft (Автокрафт) (всё ещё в разработке!)

Вкладка **AutoCraft** предоставляет мощный инструмент автоматизации для крафта предметов путем "спама" валютой (например, Chaos Orb) до тех пор, пока не будут получены нужные модификаторы.

### Обзор

AutoCraft работает путем автоматического применения выбранной валюты к предмету. После каждого применения он сканирует модификаторы предмета, чтобы проверить, соответствуют ли они вашим критериям. Если совпадение найдено, процесс немедленно останавливается.

### Начало работы

Перейдите во вкладку Smoother -> Укажите путь к .ggpk файлу -> Нажмите Load Data (это очень важно! т.к. нам надо получить информацию о модах из клиента игры)

1. **Включите модуль:** Перейдите на вкладку **AutoCraft** в главном окне POEFixer и установите галочку **"Enable AutoCraft"**.
2. **Активация в игре:**
   - Переключитесь на окно игры.
   - Наведите курсор мыши на предмет в инвентаре, который хотите скрафтить.
   - Нажмите **`Ctrl + Alt`**.
   - Откроется окно **AutoCraft Overlay**.

### Описание интерфейса (Оверлей)

Процесс крафта управляется полностью через внутриигровое окно оверлея.

*(Screenshot: AutoCraft Overlay Window)*

- **Target Item:** Отображает имя предмета, выбранного для крафта.
- **Currency:** Выпадающее меню для выбора типа валюты (например, Chaos Orb, Orb of Alteration). Инструмент автоматически определяет доступную валюту в вашем инвентаре.
- **Limit:** Функция безопасности, устанавливающая максимальное количество валюты для использования. По умолчанию 10. Бот остановится при достижении этого лимита, предотвращая случайную трату всей валюты.
- **Search Mods:**
  - Введите текст в это поле для поиска конкретных модификаторов (например, "maximum life", "resistance").
  - Ниже появится список подходящих модов. Нажмите на мод, чтобы выбрать его для настройки.
- **Mod Configuration:**
  - После выбора мода вы можете установить минимальные значения (например, если нужно минимум "+80 to Maximum Life", установите значение 80).
  - Нажмите кнопку **"+ Add"**, чтобы добавить его в список целей (Target List).
- **Target Mods List:**
  - Отображает все критерии, которые вы ищете.
  - **Логика:** Система использует логику **"ИЛИ"** (OR). Это означает, что если *любой* мод из этого списка будет найден на предмете с требуемыми значениями, процесс крафта остановится с результатом "Success".
  - Используйте кнопки "Edit" или "X" (Delete) для управления целями.

### Процесс крафта

1. **Настройка:** Убедитесь, что у вас достаточно выбранной валюты в инвентаре.
2. **Старт:** Нажмите зеленую кнопку **"START CRAFT"** в оверлее.
3. **Выполнение:**
   - Бот автоматически нажмет правой кнопкой мыши на валюту.
   - Он зажмет **Shift** для массового применения.
   - Он будет нажимать левой кнопкой мыши на целевой предмет для применения валюты.
   - После каждого клика он мгновенно считывает новые модификаторы предмета.
4. **Остановка:**
   - **Success:** Найден подходящий мод.
   - **Limit Reached:** Достигнут лимит валюты.
   - **Manual Stop:** Вы нажали кнопку "STOP".
   - **Error:** Если предмет или валюта были перемещены или потеряны.

> **Внимание:** Бот эмулирует клики мышью. Не двигайте мышью и не используйте клавиатуру во время процесса крафта (кроме нажатия горячей клавиши экстренной остановки, если настроена, или клика по Stop в интерфейсе).

### Настройки

Модуль AutoCraft предоставляет настраиваемые параметры прямо в интерфейсе:

- **Тайминги:** Настраиваемые задержки для интервала кликов, таймаута сканирования и кулдаунов действий.
- **Безопасность:** Лимит валюты за сессию, поддержка горячей клавиши экстренной остановки.
- **Поведение:** Настраиваемое удержание Shift для массового крафта, возврат курсора.

> **Экстренная остановка:** Вы можете назначить глобальную горячую клавишу для немедленной остановки процесса крафта в любой момент.

## 8. HealthBars (Полоски здоровья)

Вкладка **HealthBars** предоставляет мощную систему кастомизируемых полосок здоровья (HP) и энергетического щита (ES), отображаемых непосредственно над сущностями в игровом мире. Это помогает быстро определять приоритетные цели, следить за своим состоянием и эффективно использовать механики, такие как Culling Strike (Добивание).

### Глобальные настройки (Global Settings)

Эти настройки применяются ко всей системе или задают общие параметры.

- **Enable HealthBars:** Включение или выключение всей функции.
- **Draw in Town / Draw in Hideout:** Отображать ли полоски здоровья в мирных зонах (городах и убежище).
- **Interpolate Position:** Сглаживание движения полосок здоровья вслед за перемещением сущности.
  - **Interpolation Rate:** Скорость сглаживания (в миллисекундах). Меньшие значения делают движение более резким и точным, большие — более плавным.
- **Height Offset:** Глобальная регулировка высоты отображения полосок относительно модели сущности (полезно, если полоски висят слишком высоко или перекрывают модель).
- **Culling Strike Thresholds (Пороги Добивания):**
  - Задайте процент здоровья, при котором полоска меняет цвет, указывая, что монстр находится в диапазоне Culling Strike.
  - Настраивается отдельно для **White** (Обычных), **Magic** (Волшебных), **Rare** (Редких) и **Unique** (Уникальных) монстров.

*(Screenshot: Панель глобальных настроек)*

### Группы сущностей (Entity Groups)

Вы можете независимо настраивать полоски здоровья для разных типов существ. Это позволяет, например, сделать полоски боссов большими и заметными, а полоски обычных монстров — минималистичными или вообще скрыть их.

**Поддерживаемые группы:**

- **White Monsters:** Обычные (белые) враги.
- **Magic Monsters:** Волшебные (синие) враги.
- **Rare Monsters:** Редкие (желтые) враги.
- **Unique Monsters:** Уникальные враги и боссы.
- **Friendly Monsters:** Дружественные миньоны и NPC.
- **Player:** Ваш персонаж и другие игроки.

### Настройка группы (Per-Group Configuration)

Каждая группа имеет свой выпадающий раздел со следующими опциями:

#### Видимость и Элементы

- **Enable Group:** Включение полосок для данной группы.
- **Show Health Bar:** Отображение полоски здоровья.
- **Show ES Bar:** Отображение полоски энергетического щита (над полоской здоровья).
- **Show Culling Strike:** Изменение цвета полоски, когда цель находится в диапазоне добивания.
- **Show Text:** Отображение числового значения здоровья (например, "1.5M", "100K").
  - **Show Text Background:** Добавляет фон под текст для лучшей читаемости.

#### Цвета (Colors)

Полная настройка цветов с поддержкой прозрачности:

- **Health Color:** Цвет полоски здоровья.
- **ES Color:** Цвет полоски энергетического щита.
- **Background Color:** Цвет фона пустой части полоски.
- **Text Color:** Цвет текста.
- **Text Background:** Цвет фона текста.

#### Размеры и Позиция (Dimensions & Layout)

- **Scale (Width/Height):** Размер (ширина и высота) полоски.
- **Shift (X/Y):** Точная настройка смещения полоски относительно центра сущности (в пикселях).
- **Graduations:** Добавляет вертикальные деления на полоску для удобства оценки процента здоровья.
- **Border Thickness:** Толщина черной обводки вокруг полосок.
- **Bar Gap:** Расстояние между полоской здоровья и полоской энергощита.

## 9. Atlas (Атлас)

Вкладка **Atlas** предоставляет расширенный оверлей для внутриигрового Атласа, предлагая улучшенную визуализацию, построение маршрутов и инструменты управления для оптимизации вашей стратегии прохождения карт. Это помогает отслеживать прогресс, находить конкретные карты и планировать путь по Атласу.

### Основные возможности

#### 1. Интерактивный Оверлей

Ключевая функция — это полностью настраиваемый оверлей, который рисуется прямо поверх экрана Атласа в игре.

- **Enable Overlay:** Включает или выключает всю систему оверлея Атласа.
- **Hide When Not Focused:** Автоматически скрывает оверлей, когда окно игры не в фокусе, для экономии ресурсов.
- **Controller Mode:** Оптимизированная навигация и отображение для игроков, использующих контроллер.

*(Screenshot: Оверлей Атласа в действии)*

#### 2. Поиск и Навигация

Легко находите нужные карты или планируйте маршрут прогрессии.

- **Search Maps:** Специальная строка поиска позволяет ввести название карты (поддерживается частичное совпадение). Оверлей подсветит соответствующие карты.
- **Визуализация путей (Draw Lines):**
  - **Route Lines:** Рисует линии связей между узлами для визуализации путей.
  - **Path Thickness:** Настраиваемая толщина линий.
  - **Search Lines:** Рисует линии от вашей текущей позиции/стартового узла к картам, соответствующим поисковому запросу.
  - **Citadel/Tower Lines:** Рисует линии к важным объектам, таким как Цитадели или Башни.
  - **Range Sliders:** Настройка максимальной дальности отрисовки этих линий, чтобы не перегружать экран.

#### 3. Настройка Отображения

Фильтруйте отображаемое, чтобы сосредоточиться на текущих целях.

- **Hide Completed:** Скрывает карты, которые вы уже завершили (выполнили бонусное задание).
- **Hide Inaccessible:** Скрывает карты, до которых вы в данный момент не можете добраться.
- **Система Сетки (Grid):**
  - **Show Grid:** Накладывает сетку на Атлас для удобства ориентирования.
  - **Skip Completed:** Убирает ячейки сетки для завершенных областей.
  - **Custom Color:** Полностью настраиваемый цвет линий сетки.

#### 4. Визуальные Улучшения

Настройте внешний вид оверлея под свои предпочтения.

- **Map Badges:** Отображает иконки или текстовые значки на картах, указывающие их тир, биом или особые свойства.
- **Biome Borders:** Рисует цветную рамку вокруг карт, принадлежащих к определенным биомам.
  - **Thickness:** Настраиваемая толщина рамки.
- **Цвета и Масштабирование:**
  - **Background/Font Colors:** Установка глобальных цветов по умолчанию для элементов оверлея.
  - **Scale Multiplier:** Регулировка размера элементов оверлея (текста, иконок) относительно интерфейса игры.
  - **Anchor Nudge:** Точная настройка выравнивания оверлея, если он не идеально совпадает с окном игры.

#### 5. Управление Биомами и Контентом

Детальный контроль над подсветкой конкретных игровых механик и регионов.

- **Biome Settings:** Включение видимости и настройка цветов рамок для каждого конкретного биома.
- **Content Settings:**
  - **Badges:** Настройка подсветки для конкретных типов контента (например, Breach, Delirium, Expedition, Ritual), связанных с картами.
  - **Overrides:** Для каждого типа контента можно задать уникальные цвета фона и шрифта, чтобы выделить их.

#### 6. Группы Карт (Map Groups)

Создавайте пользовательские коллекции карт для отслеживания конкретных стратегий (например, "Фарм", "Босс-раш").

- **Custom Groups:** Создание именованных групп карт.
- **Организация:** Изменение порядка групп, их удаление или переименование.
- **Стилизация:** Назначение уникальных цветов фона и шрифта для каждой группы для быстрой визуальной идентификации на Атласе.
- **Управление:** Легкое добавление или удаление карт из группы по названию.

## 10. Hotkeys (Хоткеи / Custom Hotkeys)

Вкладка **Hotkeys** — это мощная система автоматизации и создания макросов, позволяющая настраивать сложное поведение, скрипты авто-наведения и триггеры реакции. Она разделена на два основных раздела: **Custom Hotkeys** (Пользовательские хоткеи) и **System Hotkeys** (Системные хоткеи).

### Custom Hotkeys (Пользовательские хоткеи)

Этот раздел позволяет вам создавать собственные макросы. Вы можете организовывать их в группы, задавать конкретные условия и связывать несколько действий в цепочки.

#### Группы

- **Add Group:** Создать новую папку для организации хоткеев (например, "Бой", "Баффы").
- **Toggle:** Включить или выключить целую группу сразу.
- **Rename:** Дважды щелкните по имени группы, чтобы переименовать её.

#### Настройка хоткея

Каждый хоткей состоит из идентификации, клавиши вывода/триггера, глобальных условий и списка действий.

1.  **Идентификация:**
    -   **Enabled:** Включение/выключение отдельного хоткея.
    -   **Name:** Дважды щелкните, чтобы переименовать (например, "Авто-пот", "Аим-бот").
    -   **ID:** Уникальный номер (например, #1), используемый для связывания макросов.

2.  **Глобальные условия:**
    -   **Focus:** Если отмечено, хоткей работает только когда окно игры находится на переднем плане (предотвращает случайные срабатывания при свернутой игре).
    -   **Safe Zone:** Если отмечено, хоткей отключается в городах и убежище (предотвращает пустую трату ресурсов).

3.  **Output Key (Клавиша вывода / Триггер):**
    -   Нажмите кнопку, чтобы назначить клавишу (Клавиатура, Мышь или Геймпад).
    -   **Функция:** Это клавиша, нажатие которой программа будет симулировать при выполнении действий. Она также может служить ручным триггером для действий типа "Chain".
    -   **Модификаторы:** Поддерживает Ctrl, Shift, Alt и Gamepad LT.

#### Действия (Actions)

Вы можете добавить несколько действий к одному хоткею. Они выполняются последовательно или циклично в зависимости от настроек.

-   **Repeat (Повтор):**
    -   Просто нажимает назначенную клавишу (Output Key) многократно.
    -   **Infinite:** Работает бесконечно, пока активен хоткей.
    -   **Count:** Выполняется заданное количество раз.
    -   **Interval:** Время (мс) между нажатиями.

-   **Chain (Цепочка / Связь):**
    -   Запускает *другой* хоткей по его ID.
    -   **Trigger Mode (Режим запуска):**
        -   *Program Only:* Срабатывает только если этот хоткей был запущен автоматически.
        -   *User Only:* Срабатывает только если ВЫ физически нажали клавишу.
        -   *Always:* Срабатывает в обоих случаях.
    -   **Delay:** Время ожидания перед запуском следующего хоткея.

-   **Vitals (Здоровье / Энергощит / Мана):**
    -   Нажимает клавишу, когда ваше HP/ES/Mana соответствует условию (например, `< 50%`).
    -   Полезно для авто-потов, авто-логаута или защитных умений.

-   **Buffs / Charges (Баффы / Заряды):**
    -   **Check Charges:** Срабатывает на основе количества зарядов Power/Frenzy/Endurance/Spirit.
    -   **Check Buff:** Срабатывает при наличии/отсутствии определенного баффа или проверке количества стаков/оставшегося времени.
    -   **Charged Staff:** Специальная проверка для зарядов посоха ("charged_staff_stack").

-   **Monster Count (Количество монстров):**
    -   Срабатывает, когда определенное количество монстров (с опциональным фильтром редкости) находится в заданном радиусе.
    -   Полезно для автоматического использования AoE умений в толпе.

-   **Hold (Удержание):**
    -   Удерживает клавишу нажатой в течение заданного времени (мс).

-   **MouseMove (Авто-наведение):**
    -   Самое продвинутое действие. Находит цель, наводит на неё курсор мыши и (опционально) нажимает клавишу.
    -   **Типы целей:**
        -   **Monster:** Фильтр по редкости (Normal/Magic/Rare/Unique).
        -   **Chest:** Фильтр по типу (Strongbox, Expedition, Breach и т.д.).
        -   **Player/NPC:** Выбор конкретных имен из текущей локации.
        -   **Custom:** Любая сущность по пути Metadata.
    -   **Настройки:**
        -   **Radius:** Максимальная дистанция поиска целей.
        -   **Hold Time:** Как долго удерживать прицел на цели.
        -   **Debug Mode:** Рисует красный круг (радиус) и белый текст (метаданные сущности) в оверлее.
        -   **Scan All:** Игнорирует стандартные фильтры и наводится на всё подряд (для тестов).

### System Hotkeys (Системные хоткеи)

Встроенные глобальные сочетания клавиш для управления приложением.

- **Menu Toggle:** Показать/Скрыть меню оверлея.
- **Stop Trade:** Экстренная остановка автоматизации торговли.
- **Radar - Add from Map:** Добавляет сущность под курсором в список фильтра Радара.
- **Radar - Add POI:** Добавляет Точку Интереса (POI) в месте курсора.

## 11. Launcher (Лаунчер / Limited User)

Вкладка **Launcher** предоставляет безопасную и изолированную среду для запуска игры. Она позволяет создавать и управлять "Ограниченными пользователями" (Limited Users) — учетными записями Windows с ограниченными правами — и запускать игру в контексте этих пользователей. Это особенно полезно для безопасности (Sandboxing) и запуска нескольких копий игры с раздельной конфигурацией.

### Основные возможности

#### Управление пользователями

Ядром этой вкладки является управление ограниченными учетными записями Windows.

- **Выбор пользователя:** Выпадающий список позволяет выбрать, от имени какого ограниченного пользователя будет запущена игра.
- **Создать нового (Create New):** Нажмите кнопку **"Create New"**, чтобы открыть диалоговое окно для создания нового имени пользователя и пароля. Это создает реальную учетную запись Windows в вашей системе с ограниченными правами.
- **Удалить пользователя (Delete):** Удаляет текущего выбранного пользователя из системы.
- **Управление всеми (Manage All):** Просмотр списка всех пользователей в системе. **Внимание:** Эта функция позволяет удалить *любого* пользователя, используйте её с осторожностью.

*(Screenshot: Launcher Tab Overview)*
*(Screenshot: Creating a new limited user)*

#### Настройка игры

- **Путь к игре (Game Path):** Укажите полный путь к исполняемому файлу игры (например, `PathofExile.exe` или `PathofExile_x64.exe`).
- **Обзор (...):** Используйте кнопку **"..."** для открытия проводника и выбора файла.

#### Запуск

- **Запуск игры (Launch Game):** Большая кнопка внизу запускает игру, используя учетные данные выбранного ограниченного пользователя. Приложение берет на себя сложный процесс фонового входа пользователя и безопасного запуска процесса.

### Детальное описание функций

1.  **Изоляция:** Запуск игры от имени другого пользователя изолирует процесс игры от файлов вашего основного пользователя и других запущенных процессов.
2.  **Сохранение настроек:** Лаунчер автоматически сохраняет выбранный путь к игре и предпочтения пользователя в `limitedUser/config.json`, поэтому вам не нужно вводить их каждый раз.
3.  **Безопасность:** Пароли и учетные данные управляются бэкендом приложения (`LimitedUserManager`) с использованием шифрования.

## 12. Logs (Логи)

Вкладка **Logs** — это мощный инструмент для просмотра системных событий, который позволяет отслеживать внутреннее состояние приложения, диагностировать проблемы и наблюдать за работой автоматизированных функций в реальном времени. Вкладка предлагает расширенные возможности фильтрации и управления для удобной работы с информацией.

### Обзор интерфейса

Интерфейс разделен на четыре основные области: Панель управления, Фильтры категорий, Список логов и Строка состояния.

*(Screenshot: Обзор вкладки Logs)*

### 1. Панель управления (Верхняя часть)

Расположена в верхней части вкладки и предоставляет быстрый доступ к глобальным фильтрам и управлению окном.

- **Search (Поиск):** Текстовый фильтр в реальном времени. Введите текст, чтобы мгновенно отфильтровать логи по содержанию сообщения или метке времени.
- **Level Filters (Фильтры уровня):** Кнопки-переключатели для отображения/скрытия логов по степени важности:
  - **Info:** Общие информационные сообщения (Синий).
  - **Warning:** Потенциальные проблемы или важные предупреждения (Желтый).
  - **Error:** Критические сбои или ошибки (Красный).
  - **Debug:** Глубокая отладочная информация (Серый, доступно только в Debug-сборках).
- **Category Filter Toggle (Фильтр категорий):** Нажмите на иконку фильтра, чтобы развернуть/свернуть детальную панель выбора категорий.
- **Detach/Attach Window (Открепить/Прикрепить окно):** Выносит вкладку логов в отдельное плавающее окно. Это удобно для отображения логов на втором мониторе при работе с другими вкладками основного приложения.
- **Save to File (Сохранить в файл):** Экспортирует текущую историю логов в текстовый файл в директории `logs/`. Полезно для отправки разработчикам при обращении в поддержку.

### 2. Фильтры категорий (Раскрывающаяся панель)

В развернутом состоянии этот раздел позволяет детально настроить, от каких модулей вы хотите видеть сообщения.

- **Категории:** Вы можете включить или отключить логи для конкретных функций, таких как **Trade**, **Radar**, **AutoPot**, **AutoCraft**, **Hotkeys** и др.
- **Trade Subgroups (Подгруппы торговли):** Категория **Trade** имеет дополнительные подгруппы для точной отладки:
  - *Connection:* Статус подключения к торговому сайту.
  - *Buy:* Действия покупки и операции с буфером обмена.
  - *WebSocket:* События живого потока данных.
  - *RateLimit:* Предупреждения об ограничениях API (рейт-лимитах).
  - *Icon:* Статус загрузки иконок предметов.
- **All / None (Все / Ничего):** Кнопки для быстрого включения или отключения всех категорий сразу.

*(Screenshot: Развернутые фильтры категорий)*

### 3. Список логов (Центральная часть)

Основная область отображает хронологическую историю событий.

- **Формат:** Каждая запись показывает Метку времени, Иконку уровня, Иконку категории и само Сообщение.
- **Контекстное меню:** Нажмите правой кнопкой мыши на любую запись, чтобы открыть опции копирования:
  - **Copy Line:** Скопировать выбранную строку в буфер обмена.
  - **Copy Visible:** Скопировать только отфильтрованные (видимые) логи.
  - **Copy All:** Скопировать всю историю логов.
- **Auto-Scroll (Автопрокрутка):** Список автоматически прокручивается к новым записям, если вы не прокрутили его вверх для просмотра старых логов.

### 4. Строка состояния (Нижняя часть)

Отображает статистику и инструменты очистки.

- **Счетчики:** Показывает "Total" (Всего) количество логов в памяти и "Shown" (Показано) количество, соответствующее текущим фильтрам.
- **Clear All (Очистить все):** Безвозвратно удаляет все записи логов из памяти.
- **Clear Filtered (Очистить отфильтрованное):** Удаляет только те логи, которые в данный момент видимы (проходят через активные фильтры).

## 13. Follow Bot (Бот следования)

Вкладка **Follow Bot** позволяет вашему персонажу автоматически следовать за другим игроком или сущностью. Это особенно полезно для игры на саппорте (Ауробот) или при игре в два окна (dual-boxing).

### Основные возможности

#### 1. Общее управление

- **Enable Follow Bot:** Главный переключатель для запуска/остановки бота.
- **Индикатор статуса:** Отображает текущее состояние бота (например, *Following*, *WaitingForTarget*, *Stuck*) с цветовой кодировкой для быстрой проверки.

#### 2. Настройки цели (Target Settings)

Настройка того, за кем следовать и на какой дистанции.

- **Target Name:** Вручную введите имя игрока/персонажа, за которым нужно следовать.
- **Pick Player:** Выпадающий список, показывающий игроков поблизости. Выберите одного, чтобы автоматически заполнить имя цели.
- **Stop Distance:** Дистанция, на которой бот остановится перед целью (чтобы не стоять прямо внутри модели лидера).

#### 3. Настройки передвижения (Movement Settings)

Тонкая настройка того, как бот двигается и взаимодействует с игрой.

- **Move Key:** Назначьте клавишу, используемую для передвижения (обычно это ваша клавиша "Force Move" в игре). Нажмите кнопку и затем нужную клавишу для привязки.
- **Click Ahead Distance:** Насколько далеко вперед от текущей позиции бот должен пытаться кликнуть.
- **Click Interval:** Время (в мс) между кликами передвижения.
- **Update Interval:** Как часто выполняется логика бота.
- **Path Recalc Interval:** Как часто алгоритм поиска пути пересчитывает маршрут к цели.
- **Return Cursor:** Если включено, возвращает курсор мыши в исходное положение после клика передвижения.
- **Require Focus:** Если включено, бот двигается только когда окно игры находится на переднем плане.
- **Disable in Town:** Автоматически ставит бота на паузу в городе или убежище.

#### 4. Настройки застревания (Anti-Stuck Settings)

Механизмы для обработки ситуаций, когда бот застревает.

- **Stuck Timeout:** Как долго (в мс) ждать без успешного движения перед тем, как объявить статус "Stuck".
- **Max Recovery:** Количество попыток выбраться из застревания перед тем, как сдаться или сбросить состояние.

#### 5. Настройки отладки (Debug Settings)

Визуальные инструменты для диагностики поиска пути и логики.

- **Debug Overlay:** Включает отрисовку отладочной информации на экране.
- **Show Path:** Рисует вычисленный путь к цели.
- **Show Waypoints:** Подсвечивает конкретные точки, куда кликает бот.
- **Show Stop Radius:** Визуализирует круг вокруг цели, где бот остановится.
- **Metadata Debug:** Инструменты для просмотра метаданных сущностей и тестирования поиска пути к конкретным объектам.

## 14. Debug (Отладка) [PoE 2]

Вкладка **Debug** предоставляет низкоуровневые инструменты для исследования структур памяти игры в реальном времени. Она предназначена в первую очередь для разработчиков плагинов и продвинутых пользователей, которым необходимо понимать данные сущностей, значения компонентов и иерархию UI-элементов.

### 1. Список сущностей (Entity List)

Фильтруемая таблица, показывающая все сущности, загруженные в текущей зоне.

- **Столбцы:** Entity ID, Адрес, Путь метаданных, Тип, Подтип, Состояние, Редкость, Зона (InnerCircle/OuterCircle/Far).
- **Фильтры по типу:** Быстрая фильтрация по типу сущности — Монстры, Сундуки, NPC, Игроки, Предметы и др.
- **Поиск:** Текстовый поиск по пути метаданных.
- **Счётчик:** Отображает общее количество сущностей и отфильтрованное количество.

### 2. Наблюдение за сущностью (Entity Watch / Component Inspector)

Нажмите на сущность в списке, чтобы начать "наблюдение" за ней. Рабочий поток приложения будет считывать подробные данные компонентов этой сущности каждый кадр.

**Доступные компоненты:**
- **Life:** Текущие/Максимальные HP, ES, MP, зарезервированные значения, скорость восстановления.
- **Render:** Позиция в мире (X/Y/Z), границы модели, имя/путь модели.
- **Positioned:** Позиция на сетке, вращение, масштаб.
- **Targetable:** Флаги IsTargetable, IsHostile, IsFriendly.
- **Animated:** Текущий путь анимации, ID анимации.
- **Stats:** Полный словарь характеристик (ID характеристики → значение).
- **Actor:** Список активных умений, текущий ID действия.
- **Buffs:** Все активные баффы с именем, зарядами, оставшимся временем, общей продолжительностью.

- **Все компоненты:** Отображает полный список всех пар имя→адрес компонентов, даже для компонентов без специализированного парсинга (полезно для продвинутого чтения памяти).

### 3. Отладка инвентаря (Inventory Debug)

Детальная инспекция системы инвентаря игры.

- **ServerData:** Отображает адрес компонента ServerData.
- **Список инвентарей:** Показывает все инвентари игрока (ID, Адрес, Название).
- **Наблюдение за инвентарём:** Выберите инвентарь для детальной проверки:
  - **Сетка слотов:** Визуальное представление занятых/свободных слотов.
  - **Список предметов:** Все предметы с путём метаданных, редкостью, позицией в слоте.
  - **Моды предметов:** Полная разбивка модов (Неявные, Явные, Зачарование, Hellscape) со значениями.

### 4. UI Explorer (Проводник интерфейса)

Навигация по дереву иерархии UI-элементов игры.

- **Game UI Root:** Инспекция элементов под корнем игрового UI (панели, меню, кнопки).
- **Top-Level UI Root:** Доступ к верхнеуровневому корню UI для экранов логина/загрузки.
- **Навигация по дереву:** Разворачивание/сворачивание UI-элементов для просмотра дочерних элементов, адресов и свойств.
- **Поиск:** Фильтрация элементов по имени/пути.
- **GameCullValue:** Отображается для расчётов масштаба UI.

## 15. Плагины (Plugins)

Вкладка **Plugins** управляет нативными C++ DLL-плагинами, расширяющими функциональность POEFixer. Плагины могут отрисовывать оверлеи, получать доступ к игровым данным и интегрироваться с существующим интерфейсом.

### Управление плагинами

- **Список плагинов:** Отображает все обнаруженные плагины из директории `Plugins/`.
- **Включение/Отключение:** Переключение отдельных плагинов. Состояние сохраняется между сессиями.
- **Настройки плагинов:** Каждый включённый плагин может отображать собственный раздел настроек прямо во вкладке Plugins.
- **Проверка версии SDK:** Хост проверяет совместимость версии SDK плагина при загрузке. Текущая версия SDK: **4**.

### Доступные плагины

#### Kill Counter (Счётчик убийств)

Комплексный оверлей для отслеживания убийств, сундуков и смертей.

- **Подсчёт убийств монстров:** Считает убитых монстров по редкости (Normal, Magic, Rare, Unique, Unknown) используя детекцию по исчезновению.
- **Подсчёт сундуков:** Считает открытые сундуки по типу (Magic, Rare, Expedition, Breach, Strongbox).
- **Счётчик смертей:** Отслеживает смерти игрока.
- **Счётчики по карте:** Автоматический сброс при входе на новую локацию и убийстве первого монстра (старые счётчики видны до первого убийства на новой локации).
- **Общие итоги:** Постоянная статистика, хранящаяся в базе данных SQLite3.
- **Оверлей:**
  - Перемещаемый при активном меню программы (появляется подсказка "Drag to reposition").
  - Неинтерактивный и без заголовка, когда меню скрыто.
  - Настраиваемая прозрачность (0-100%), изменяемый размер.
  - Вкладки Map и Total для переключения между текущей картой и общими итогами.
- **Атлас иконок:** Отображает иконки типов сущностей из общего спрайт-листа `Resources/radar/icons.png`.
- **Настройки:** Выбор отображаемых счётчиков, прозрачность, режим оверлея, кнопки сброса.

#### Example Plugin (Пример плагина)

Эталонная реализация, демонстрирующая все возможности Plugin SDK:

- **Инспектор баффов:** Список баффов игрока с фильтрацией и прогресс-барами.
- **Исследователь сущностей:** Список сущностей с механизмом наблюдения и деревом компонентов.
- **Инспектор инвентаря:** Просмотр ServerData, выбор инвентаря, сетка слотов, моды предметов.
- **Просмотр памяти:** Hex-дамп, демо `Read<T>`, сканер паттернов.
- **UI Explorer:** Полный навигатор дерева UI-элементов с поиском и подсветкой.
